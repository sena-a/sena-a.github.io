테마 수정 삽질기

우선 테마를 만드신 개발자님의 documentation에 따르면, 테마를 그대로 쓰지 않고 수정하고 싶다면 아래 방법을 통하면 된다고,

이렇게 아주 간단하고도 친절하게 적어주셨는데, hexo랑도, node.js랑도 아직 서먹한 나는 이것만 보고 따라했다가 여러 삽질을 하다가, 하다가, 또 하다가.. 거의 나무 100그루는 심었다. 우선 이 테마는 css를 수정할 때 파일만 수정해서 저장하면 바로 적용되는게 아니라 빌드 과정을 거쳐야지 비로소 우리의 블로그에 장착되게 되어있다. 왜죠? 암튼, 그래서 수정하기 전에 npm run start를 하고 신나게 수정했다. 수정을 하고, npm run prod를 했는데 여기서부터 나의 땅파기가 시작된다.

분명 나는 제대로 수정을 다 했는데, npm run start하고 수정하고 npm run prod도 했는데, 빌드만하면 내가 넣은 이미지고 수정한 코드고 다 백투더파스트하는게 아니겠는가? 첫 번째는 아이 뭐 이럴 수도 있지 하하, 하고 다시 도즈언해봤다. 또 모든게 사라졌다. 하… 그런데 아무리 구글링을 해봐도 나같은 사례가 없어서 더 우울했다. 개발 쪼렙의 설움이 폭발하면서 오기가 생겼다. documentation을 읽고 읽고 또 읽었다. 모르겠다. 그래서 한 번 더 시도해봤다. 역시 날 실망시키지 않고 백투더파스트해서 수정한 코드는 다시 처음 그 상태로 돌아간다. 도르마무랑 거래하는 줄… 테마 이슈에 들어가서 뒤져봤는데 역시 해결은 되지 않았다.

테마 폴더 삭제하고 다시 깔아보기도 하고 쌩삽질을 하면서 101번째 나무 심으려다가 너무 미련한 것 같아서 차분한 마음으로 _config.yml을 처음부터 다시 읽어봤다.

…?…?

친절하게 잘 적어놓아주셨는데도 불구하고 나는 그 친절함을 짓밟고 혼자 삽질하다가 이런 벌을 받았다.. 멍청하게도 assets 폴더에 이미지를 넣고, assets 폴더의 파일들만 건드리면서 왜 없어지냐고 유령이냐고 삽질하고 있었다. _images, _css 폴더에서 수정하고 추가하니까 모든게 평화롭더라. 그러니까 역시 차분하게 설명서를 꼼꼼히 읽는것이 정말정말정!말! 중요하다는 것을 다시 느꼈다.

테마 수정 요약

아, 수정하면서 grunt라는 빌드 툴에 대해서 알아볼 수 있었다. 이 테마는 npm run start를 하면 npm run grunt -- default가 실행되게끔 스크립트가 설정되어있는데, 이 명령어가 실행되면 뭐가 쫘라락 뜨고 마지막에

이 상태로 그대로 정지하게 된다. 처음엔 이게 뭔가 했는데, 찾아보니 watch는 files에 지정된 파일들의 변경사항을 감시하다가 변경사항이 발생하면 tasks에 지정된 동작을 실행한다고한다. waiting…에서 넘어갈 줄 알았는데 안 넘어가서 당황했었는데 찾아보니 이해가 됐다. 아무튼.. 수정이 되어서 정말 다행이닿.
