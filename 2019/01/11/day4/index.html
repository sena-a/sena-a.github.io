day4 퀴즈

  1. function Person(){}, const person = Person(), const person = new Person() 의 차이점은 무엇입니까?

전자는 단순히 변수에 함수 값을 대입한 것이고, 후자는 Person() constructor가 person이란 이름의 인스턴스가 생성된 것 입니다.

  1. 호스트 객체와 내장 객체의 차이점은 무엇입니까?

내장 객체는 JavaScript에 미리 생성되어있는 객체고, JavsScript 언어의 일부입니다.
호스트 객체는 런타임 환경에서 (브라우저 혹은 노드)에서 제공되는 객체입니다.

  1. 객체 속성을 반복할 때(또는 나열할 때) 사용하는 언어 구조는 무엇입니까?

단순히 for문으로 객체의 속성을 나열할 때는 상속받는 속성도 나열되기 때문에, getOwnPropertyNames() 메소드를 쓰는 것이 좋습니다.

getOwnPropertyNames()은 전달하는 객체의 열거 가능한 속성과 열거되지 않는 모든 속성을 나열하는 정적 메서드

  1. 프로토타입 상속이 어떻게 작동하는지 설명하세요.

프로토타입 상속은 실제 상속을 받았다기 보다는 연결되어있는 프로토타입에 존재하는 속성을 공유할 수 있기 때문에 이것을 상속이라고 표현하는 것인데, 해당 객체에 불러온 속성이 없으면 연결된 프로토타입 객체로 가서 해당 속성을 찾아보고, 또 속성이 없으면 그 프로토타입 객체에 연결된 프로토타입 객체로 가서 속성을 찾아보는 방식이며, 이를 프로토타입 체인이라고 합니다.

  1. 내장 객체 확장의 장단점

원하는 메소드를 어디서든 편하게 쓸 수 있게 되지만, 내장 객체에 이미 존재하는 메소드를 덮어쓸 수 있고, 결국에는 원하는 대로 메소드가 동작하지 않을 수 있다는 매우 큰 위험이 있습니다.
